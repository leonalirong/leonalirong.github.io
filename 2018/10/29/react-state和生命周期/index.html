一、props && state

1. React元素 （虚拟DOM元素 | js代码中的任意标签 | 通过React。createElement 创建出来的js对象）

  2. 如何区分React 元素与React组件的概念: 组件就是一系列React元素的组成
  3. 什么是元素变量：定义一个变量，变量的值是React元素
  4. React 元素是一个不可变的对象
  5. 组件默认prop值:

2. State

  1. 数据绑定的方式有两种：
    props
    state data 私有的

  2. 有状态组件？无状态组件？
    a. 组件有没有状态就看这个组件有没有state
    b. 一般类组件就可以称为有状态，函数组件就叫无状态组件

  3. 如让Clock组件自己去更新ＵＩ，首先需要让这个组件有自己的state，然后state如果发生变化，他就会重新渲染

  4. 组件要有变化，两种方式：
    a. 组件接收的props有变化
    b. 组件自身的state有变化

  5. 组件的state或者props有变化，组件就会更新，更新的效果其实就是类组件的render函数重新执行了，函数组件重新调用了

    如果某个数据不需要重新更新，那么就不要把它写在构造函数的state里面（或者props里面）

3. 总结：

  1. 如果要让组件有state，首先要确保这个组件是类组件
  2. 要在构造函数中定义状态的初始值
  3. 写了构造函数的话，就必须在构造函数的第一行调用父类的super
  4. 不能直接修改this.state里的值，而是要用this.setstate({ })方法

4.点击的时候调用方法，为何要用bind而不是call和apply方法

call 和 apply以及 bind 方法都可以实现继承，
事实上apply 和 call 的用法几乎相同, 唯一的差别在于：
当函数需要传递多个变量时, apply 可以接受一个数组作为参数输入, call 则是接受一系列的单独变量。

bind方法：和call很相似，第一个参数是this的指向，从第二个参数开始是接收的参数列表。区别在于bind方法返回值是函数以及bind接收的参数列表的使用。
bind 方法不会立即执行，而是返回一个改变了上下文 this 后的函数

4. PureComponent 提高运行效率

1. PureCompoent 会在组件内容，比较数据是否有更新，如果有就render。如果没有就不会 render

  1. 父组件 render 了，那么子组件一定会 render

2. PureComponent 组件内事件函数绑定了 bind 的话，bind会生成一个新的数组，所以即使在PureComponent组件内，依然会将没有任何改变的子组件进行渲染
